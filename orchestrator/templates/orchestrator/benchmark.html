Prompt categories
Each category uses fixed prompts and a max output size. Every selected category is tested at each concurrency level.
  • Short — ~20 output tokens (latency baseline)
  • Medium — ~100 tokens (typical chat turn)
  • Long — ~400 tokens (sustained generation stress)
Concurrency — max parallel requests per scenario. 1 = single user; 4 = four clients at once. Comma-separated values run separate scenarios (e.g. 1,4,8).
Requests / scenario — total requests per (category × concurrency) pair. Use 20–50 for stable p95 estimates; 5 is fine for a quick check.

Result metrics

TTFT p50
Time To First Token — delay until the first streamed token arrives. Lower is better for interactive UX.
Latency p50 / p95
End-to-end time until the full response is received. p95 shows tail latency under load.
Tok/s (per request)
Output tokens per second for a single request — useful to compare decoding speed in isolation.
Aggregate tok/s
Total output tokens divided by wall time — best metric for server throughput under concurrency.
